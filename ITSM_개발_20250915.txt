service_requests 테이블의 데이터는
일반사용자, 배정담당자, 조치담당자 정보는 동명이인 중복을 방지하기 위해 각각 ID로 매칭 되며,
성명과 소속명은 생성 시 정보를 보존하고 검색 등에 활용한다.
또한 소속명은 List및 통계 등 조회작업 시 활용 한다.




1. 권한
: 기존소스에 적용 된 권한은 다시 조정한다.
- 일반사용자       ← 일반 권한 = 서비스 신청 및 일반문의 사항을 신청이나 등록, 조회 하는 권한
- 배정담당자       ← 기술자 권한 = 서비스 신청의 조치 담당자 배정 및 일반문의 사항을 답변하는 권한
- 조치담당자       ← 기술자 권한 = 배정 된 작업을 조치 하는 권한
- 관리매니저       ← 서비스관리 권한 = 배정담당자 및 소속의 조치담당자들의 배정 및 작업 현황을 관리 하는 권한
- 시스템관리       ← 시스템관리 권한 = 모든 사용자 및 기술자 회원정보를 관리하는 권한

2. 흐름도(프로세스)
: Figma로 구성 되어 있으며 간략한 흐름도를 설명하며 자세한 사항은 피그마 파일을 참고하기 바란다.
- (일반사용자) 서비스 신청 → (배정담당자) 조치 담당 배정 → (조치담당자) 작업 업무 진행 및 완료 절차로 진행한다.
- (일반사용지) 일반문의 등록 → (배정담당자) 일반문의 답변으로 진행한다.
- (일반사용자) 자주하는질문은 조회 권한을 가진다.
- (관리매니저) 관리매니저 소속 조치담당자들의 업무를 서비스 신청에서 부터 작업 업무 완료까지 조회, 수정, 삭제 관리 권한을 가지며, 일반문의사항에 대해서 도 조회, 수정, 삭제의 관리 권한을 가진다. 또한 자주하는 질문에 대해는 조회, 등록, 수정, 삭제 관리 권한으로 진행한다.
- (시스템관리) 모든 관리매니저의 업무 관리 권한을 가지며, 시스템 관리한다.

3. 메인 페이지
3-1. 레이아웃 구성
: 메인페이지는 일반사용자 관점으로 설계 하였으며 중앙부분에 일반사용자가 클릭 할 수 있는 4가지 항목를 배치한다.
: 기술자 및 서비스, 시스템 권한은 우측 상단에 기술자 로그인을 배치한다.
3-2. 메인 페이지 클릭 항목
: 페이지에 배치 된 각 항목을 클릭하면 일반사용자 로그인 프레임이 오버레이 되고 로그인 후 각 항목의 페이지로 이동한다.
      항목                              이동 페이지
- 자주하는 질문        → 자주하는질문 페이지로 이동             
- 요청 진행사항        → 요청진행사항 페이지로 이동
- 서비스 신청           → 서비스 신청 페이지로 이동
- 일반 문의사항        → 일반문의사항 페이지로 이동

4. 로그인 프레임
4-1. 레이아웃 구성
: 오버레이로 프레임이 열리고, 일반사용자를 위해 로그인 상태 유지 Check Box를 구성하고 보안에 대해 문제가 없을 경우 적용한다. 
4-2. 일반사용자 로그인
: Figma로 작성 된 파일에 로그인 프레임이 각 항목별로 각각 구성 되어 있지만 실제로는 일원화 하여 1개의 오버레이 프레임으로 구성하고 로그인 후 선택 된 각 페이지로 이동하도록 소스에 반영한다.
4-3. 기술자및 서비스관리, 시스템관리 로그인
: 메인 페이지 우측 상단에 배치 된 Tech login 버튼을 클릭하면 기술자및 서비스관리, 시스템관리 권한의 로그인 프레임이 오버레이로 나타나고 로그인 된 상태에서 각 권한별 페이지로 이동한다. 이 또한 로그인 프레임 일원화를 위해 일반사용자 로그인 프레임을 활용하여 1개의 로그인 프레임으로 관리 되도록 소스에 반영한다.
- 배정담당자 페이지 : 서비스 신청 된 데이터의 조치담당자를 배정하는 업무를 진행하고, 일반 문의에 등록 된 답변 업무도 진행한다. 
- 조치담당자 페이지 : 배정담당자에 의해 배정 된 서비스 신청 정보를 조회하고 배정작업 승인 및 반려를 진행하고 배정작업 승인 후에는 서비스 작업 진행을 수행한다.
- 관리매니저 페이지 : 배정담당자 및 소속 조치담당자들의 업무를 관리한다. 
- 시스템관리 페이지 : 시스템관리를 위해 관리매니저의 업무 권한과 회원 정보 관리한다.
4-3. 비밀번호
: 비밀번호 찾기 기능은 보안을 고려하여 진행 되도록 한다.
특히. 비밀번호는 보안이 적용 되도록 구성 되어야 한다.

5. 회원가입 페이지
: 회원가입이 않된 경우 회원가입을 진행 되도록 해야 한다. 모든 사용자가 가입하고 권한 조정은 시스템관리에서 조정 관리한다.
5-1. 레이아웃 구성
: 로그인 프레임에서 회원가입을 진행하여 오버레이로 회원가입 프레임이 열린다.
5-2. 회원가입 입력 Column
: 회원가입의 "*"는 반드시 작성하도록 소스에 반영해야 한다.
   Column             처리
- 이메일(ID)*      ← (입력)
- 비밀번호*       ← (입력) 반드시 보안이 적용 되도록 구성
- 성명*             ← (입력)
- 직급*             ← (입력)
- 소속*             ← (입력)
- 연락처*           ← (입력)
- 생성일자        ← (자동생성) 표기 되지 않으며 회원가입이 완료 될때 현재시간을 반영
- 권한              ← (자동생성) 표기 되지 않으며 회원가입이 완료 될때 기본값 "일반사용자"로 권한을 갖도록 반영(권한은 시스템관리에 의해 권한 조정 관리된다.)

6. 자주하는질문 페이지
6-1. 레이아웃 구성
: 일반사용자는 OA 관련 서비스를 이용 시에 빈번히 발생되는 문제 등을 일반적으로 조치 할 수 있는 정보를 조회 하도록 설계하고, 표기 항목에는 발생 원인요약, 발생 원인내용, 구분 등의 정보를 각각의 원인별로 나타내고 각항목을 클릭 시 자주하는질문 조회 프레임을 오버레이로 열어준다. 자주하는질문의 관리 권한은 관리매니저가 가진다.
- 자주하는질문은 페이지는 6개 Row 항목씩 나타나게 하고 6건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.
- 자주하는질문 조회 프레임은 아이콘, 발생 원인요약, 발생 원인내용, 즉시해결, 문제가 지속될 경우 등의 정보를 열어준다.
- 헤더 메뉴 부분에 각 페이지로 이동 할 수 있도록 "요청진행사항", "서비스신청", "일반문의사항"를 배치하고, 우측 상단에 "로그아웃"버튼도 배치한다.
6-2. 자주하는질문 항목및 내용 관리권한
: 자주하는질문 수정 페이지의 추가, 수정 및 삭제 등은 관리매니저 페이지에서 관리한다.

7. 요청 진행사항 페이지
7-1. 레이아웃 구성
: 일반사용자 자신의 서비스 신청 정보만 List로 조회 하도록 나타낸다.
- List에 나타내는 Coulmn으로는 신청번호, 신청제목, 진행, 신청일시, 담당자, 완료일시 등의 Column으로 구성한다.
- 좌측 List 제목 위쪽에 검색 기간을 배치하고 기본값은 금일 기준이고, 선택 된 검색 기간내에 자신의 서비스 신청 정보만 조회하도록 List에 반영한다.
- List의 Record가 10건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.
- 헤더 메뉴 부분에 각 페이지로 이동 할 수 있도록 "서비스신청", "자주하는질문", "일반문의사항"를 배치하고, 우측 상단에 "로그아웃"버튼도 배치한다.
- 좌측 상단에 새로고침 버튼을 배치하고 클릭 시 초기화 하여 List를 현재기준으로 나타낸다.
7-2. 진행 Column 값 매칭         
: Database의 서비스 작업처리 Table에 단계 Column은 접수, 배정, 재배정, 확인, 예정, 작업, 완료, 미결 등으로 진행사항에 따라 값을 가지며, 이를 매칭하여 요청 진행사항 페이지의 진행 Column에 매칭된 값을 보여준다.
  (진행 Column 매칭 값)       매칭         (단계 Column 값)
- 정상접수                          ←           접수
- 담당배정                          ←           배정, 재배정, 확인
- 시간조율                          ←           예정
- 작업진행                          ←           작업
- 처리완료                          ←           완료
- 미결처리                          ←           미결
7-3. 일반사용자 서비스 신청 정보 List
- 진행 Column이 "정상접수"인 경우만 서비스 신청 정보 List에 수정,삭제 버튼을 활성화 하고, 그외 진행 Column은 조회만 가능하도록 상세보기 버튼을 List에 활성화 한다.
7-4. 신청정보수정 프레임
: 서비스 신청 정보 List에서 수정 버튼을 클릭하면 신청번호, 신청자, 신청연락처, 신청위치, 신청일시, 실제신청자, 실제연락처 등 Cloumn의 값을 조회 되도록 나타내고 신청제목, 신청내용, 현재상태 만 수정 가능 하도록 신청정보수정 오버레이 프레임을 열어주고 하단에 수정완료 버튼을 배치한다.
7-5. 신청정보삭제 프레임
: 서비스 신청 정보 List에서 삭제 버튼을 클릭하면 신청번호, 신청제목, 신청내용, 신청자, 신청연락처, 신청위치, 신청일시, 현재상태, 실제신청자, 실제연락처 등 Cloumn의 값을 조회 되도록 신청정보삭제 오버레이 프레임을 열어주고 하단에 삭제완료 버튼을 배치하고, Figma파일에는 적용이 되어 있지는 않지만 삭제완료 여부를 다시 한번 확인하는 절차도 소스에 반영하도록 구성한다.
7-6. 서비스상세보기 프레임
: 서비스 신청 정보 List에서 삭제 버튼을 클릭하면 서비스 신청정보 프레임과 작업 진행정보 프레임을 1개의 오버레이로 열어주고, 서비스 신청 정보 프레임에는 신청번호, 신청제목, 신청내용, 신청자, 신청연락처, 신청위치, 신청일시, 현재상태, 실제신청자, 실제연락처 등 Column의 값을 조회 되도록 나타내고 작업 진행 정보는 배정일시, 서비스유형, 조치담당자, 예정조율일시, 작업시작일시, 작업 내역, 작업완료일시, 문제사항, 미결완료 Check Box 등 Columm의 값을 조회 되도록 나타내고, 하단에 나가기 버튼을 배치한다.
7-7. 정보변경 프레임
: 요청진행사항 페이지의 우측 메뉴 헤더에 배치 된 정보변경은 자신의 회원정보를 수정하도록 구성한다. 반드시 비밀번호 변경 시 기존 비밀번호 확인 후 변경하도록 구성한다.

8. 서비스 신청 페이지
8-1. 레이아웃 구성                    
: 일반사용자가 OA관련 서비스를 신청하는 정보를 입력하는 페이지로 구성하고, 서비스 신청 정보를 입력한 후 신청하기 완료 되면 요청진행사항 페이지로 이동한다.
- 헤더 메뉴 부분에 각 페이지로 이동 할 수 있도록 "요청진행사항", "자주하는질문", "일반문의사항"를 배치하고, 우측 상단에 "로그아웃"버튼도 배치한다.
8-2. 서비스 신청의 입력 되는 Column
: 배정담당자의 작업배정에서 관리 되는 모든 Column과 조치담당자의 배정확인 및 배정반려, 작업정보등록에서 관리 되는 모든 Column를 참고하여 Database Table를 구성 해야한다.
   Column                  처리
- 신청일시              ← (자동생성) 현재시간을 반영
- 신청번호              ← (자동생성) 표기 되지 않으며, 신청하기 버튼을 클릭하면 발급 규칙을 적용하여 발급(앞자리 기본 "SR"을 붙이고 날짜와 순서로 구성한다. 예, SR-20250831-001)
- 현재상태              ← (선택) 드롭다운으로 데이터 값을 정상작동, 오류발생, 메시지창, 부분불능, 전체불능, 점검요청, 기타상태 로 7가지를 선택 하도록 구성
- 관리번호              ← (입력)
- 신청위치              ← (입력)
- 신청자                 ← (자동생성) 자신의 성명을 가져옴
- 신청연락처           ← (자동생성) 자신의 소속을 가져옴
- 신청소속              ← (자동생성) 자신의 소속을 가져옴
- 대리신청여부        ← (선택) 체크 박스로 구성하고 대리신청 인 경우는 실제신청, 실제연락처, 실제소속 등 Column을 입력하도록 활성화 함

- 실제연락처           ← (입력) 대리신청 여부 Check 된 경우 표시 및 입력 가능
- 실제소속              ← (입력) 대리신청 여부 Check 된 경우 표시 및 입력 가능

9. 일반문의사항 페이지
9-1. 레이아웃 구성
: 일반사용자 자신의 일반문의사항에 문의 된 사항만 List로 조회 하도록 나타낸다.
- List에 나타내는 Column으로는 문의일시, 문의내용, 문의자, 답변일시, 잠금아이콘 등으로 구성한다.
- List에 문의자 Column 표기 시 자신의 성명은 Full 보여주고 그외 문의자는 성만 표기하고 나머지는 "*"표기 한다. 예) "김**"와 같이 보안 적용한다.
- 비밀글여부 Column에 Check 된 경우는 시각적으로 잠금이 되어 있음을 나타내고, 비밀글여부 Column에 Check가 되지 않는 경우는 시각적으로 잠금이 되어 있지 않음을 나타낸다. 또한 비밀글여부 Column에 Check가 되지 않고 답변일시 Column에 데이터값이 있는 답변보기 버튼을 활성화 하고 문의 답변 정보를 조회 가능 하도록 답변보기 프레임을 오버레이로 나타내도록 구성한다.
- 우측 List 제목 위쪽에 검색 기간을 배치하고 기본값은 금일 기준이고, 선택 된 검색 기간내에 자신의 일반문의사항만 조회하도록 List에 반영한다.
- List의 Record가 10건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.
- "작성글만보기" 버튼를 List 제목 위쪽에 배치하고 버튼을 클릭하면 자신의 문의 작성글 List만 나타낸다.
- 좌측 List 제목 위쪽에는 "문의글쓰기" 버튼을 배치하고 자신의 일반문의사항을 등록 할 수 있도록 문의글쓰기 프레임을 오버레이로 열어준다.
- 좌측 상단에 새로고침 버튼을 배치하고 클릭 시 초기화 하여 List를 현재기준으로 나타낸다.
9-2. 문의글쓰기 프레임에 입력 되는 Column

   Column              처리

- 문의자            ← (자동생성) 자신의 성명을 가져옴
- 문의내용         ← (입력)

: List에서 답변보기 버튼을 클릭하면 해당 되는 답변보기 프레임을 오버레이로 열어 조회 하도록 구성하고 문의일시, 문의자, 문의내용, 답변내용, 답변자 등의 Cloumn 값을 나타낸다.
- 문의자 Colmn 표기 시 자신의 성명은 Full 보여주고 그외 문의자는 성만 표기하고 나머지는 "*"표기 한다. 예) "김**"와 같이 보안 적용한다.

: 배정담당업무 페이지 좌측에 조치담당자 작업현황 프레임을 구성하고 List의 Column은 조치담당자, 작업, 배정건수 등을 나타낸다.
- 조치담당자별 금일 기준으로 "배정" 중인 건수를 실시간 취합하여 List로 표기하고 조치담당자의 작업 가능여부를 List의 작업 Column에 표기한다.

- 조치담당자, 작업가능여부, 배정진행건수를 실시간 조회 할 수 있도록 한다.

10-12. Toggle

- 조치담당자 작업현황 프레임의 Toggle은 조치담당자 작업현황을 On, Off 할 수 있도록 한다. 조치담당자 작업현황 Toggle이 On인 경우는 페이지에 나타내고, Off 인 경우는 나타내지 않는다.

11. 조치담당자 페이지
11-1. 레이아웃 구성

11-2. 조치담당자 작업 List 프레임
: 조치담당자 작업 List의 Column은 신청번호, 신청제목, 현재상태, 신청자, 신청소속, 배정시간, 단계, 서비스유형, 완료일시 등을 나타낸다.
- 우측 List 제목 위쪽에 검색 기간을 배치하고 기본값은 금일 기준이고, 선택 된 검색 기간내에 조회하도록 List에 반영한다.
- List의 Record가 10건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.  
- 우측 List 제목 위쪽에 "미 완료 조회" 버튼을 배치하고 작업완료일시 Column에 값이 없는 Record 만 List에 나타낸다.
- 단계 Column에 "배정"으로 된 경우는 배정확인 버튼을 활성화 하고, 그외 작업완료일시 Column에 값이 없는 Record는 정보확인 버튼을 활성화 한다.
- 각각의 배정확인 버튼과 정보확인 버튼을 클릭하면 오버레이로 각각의 배정확인과 정보확인 프레임을 열어준다.
- 정보확인 버튼을 클릭하면 2개의 프레임으로 진행되며, 첫번째는 서비스 신청 정보를 조회 하도록 하고 작업정보등록을 두번째 프레임인 나타나 작업정보등록 할 수 있도록 한다.

- 조치담당자의 작업 진행 절차는 단계적으로 진행 되도록 소스에 반영하고 각단계별 예정조율일시 → 작업시작일시 → 작업완료일시, 작업내역 Column에 입력하고 처리 버튼을 눌러 다음 단계를 진행하고 미해결 완료가 발생한 경우는 작업완료일시 처리 단계까지 진행하고 문제사항을 입력 후 등재 버튼을 클릭하고 미결 완료 Check Box를 클릭하여 등재 버튼을 눌러 해당 Column에 반영한다.

- 단계 Column의 "완료" 단계은 작업시작일시가 처리 된 상태에서 작업 내역을 입력하고 처리 버튼을 클릭하고 작업 완료 일자와 시각을 작업 완료 일시 Column에 입력하고 처리 버튼을 클리하면 단계 Column에 "완료"로 값을 가진다.

12-2. 서비스 작업 List 관리 프레임
: 서비스 작업 List의 Column은 신청번호, 신청시간, 신청제목, 현재상태, 신청자, 신청소속, 배정시간, 단계, 조치자, 조치소속 등을 오버레이 프레임으로 나타낸다.
- 초기의 List에 나타낼 Record는 두가지를 일자별 시간순으로 조합해서 나타낸다. 첫번째는 현재기준 단계 Column에 "접수", "재배정"인 경우는 조치담당자가 확정 되지 않았으므로 List에 나타내고 각각 "배정작업"과 "재배정작업" 버튼을 활성화 한다. 두번째는 관리매니저가 소속별 권한을 가지고 있어 자신이 속한 소속의 조치담당자 현황 만 List로 나타내며, 수정 및 삭제 버튼을 활성화 한다.
- List에 접수시간 Coulmn의 값은 신청일시 Column의 값 중 시간 부분만 발취하여 표기한다. 예) 13:00
- List에 배정시간 Coulmn의 값은 배정일시 Column의 값 중 시간 부분만 발취하여 표기한다. 예) 13:10
- 우측 List 제목 위쪽에 검색 기간을 배치하고 기본값은 금일 기준이고, 선택 된 검색 기간내에 조회하도록 List에 반영한다.
- List의 Record가 10건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.
- 우측 List 제목 위쪽에 "미결 완료 조회" 버튼을 배치하고 클릭 시 완료일시 Column에 값이 없는 Record 만 List에 나타낸다.
- 검색조건으로 "부서별" 드롭다운 선택 할 수 있도록 구성하고 검색조건에 따라 부서별 조치담당자를 List로 나타낸다. 단. 소속의 조치담당자가 아닌 경우는 수정 및 삭제 버튼을 활성화 하지 않는다.
- 좌측 상단에 새로고침 버튼을 배치하고 클릭 시 초기화 하여 List를 현재기준으로 나타낸다.
12-3. 배정작업 및 재배정작업
: 관리매니저는 배정담당자의 권한도 가지고 있어 List의 "배정작업"과 "재배정작업" 버튼을 각각 클릭하면 해당하는 "10-3. 배정작업 프레임", "10-4. 배정정보에 입력 되는 Column"과 "10-5. 재배정작업 프레임", "10-6. 재배정정보에 입력 되는 Column"를 배정담당자와 동일 권한으로 관리한다.
12-4. 작업정보관리 프레임
: 관리매니저는 소속 조치담당자들의 업무를 관리 할 권한을 가지고 있어 작업정보를 수정 할 수 있어 작업정보관리는 두가지 프레임을 조합해서 1개의 오버레이로 나타낸다. 첫번째는 좌측에 서비스 신청 정보을 확인할 수 있도록 "11-6. 정보확인 프레임"과 같이 신청번호, 신청제목, 신청내용, 신청자, 신청연락처, 신청위치, 신청일시, 현재상태, 실제신청자, 실제연락처 등 Column를 조회 하도록 나타내고 버튼은 없으며, 두번째 우측에 "11-7. 작업정보등록 프레임"과 같이 배정일시, 배정담당자, 배정의견, 서비스유형, 조치담당자 Column의 데이터값을 조회 정보로 나타내고 작업정보를 단계적으로 수정 할 수 있도록 하고 하단에 수정완료 버튼을 배치한다.
- 수정 프로세스는 조치담당자의 작업 진행 절차를 참고해서 수정이 가능하도록 소스에 반영한다.
12-5. 작업정보삭제 프레임
: 관리매니저는 소속 조치담당자로 배정 된 서비스 신청을 정보를 삭제 할 권한을 가지고 있으며, 작업정보관리와 같이 두가지 프레임을 조합해서 1개의 오버레이로 나타내고 데이터 값은 모두 조회 처리하고 하단에 삭제하기 버튼을 배치한다.
- 삭제하기 버튼을 클릭하면 한번 더 삭제 의사를 물어 보는 절차를 소스에 반영한다.
12-6. 자주하는 질문 관리 프레임
: 관리매니저는 자주하는 질문을 관리하는 권한이 가지며, 상단에는 자주하는질문항목을 추가할 수 있는 질문추가와 질문삭제 버튼을 배치하고 표기 항목에는 발생 원인요약, 발생 원인내용, 구분 등의 정보를 각각의 원인별로 나타내고 각항목을 클릭 시 자주하는질문 수정 프레임은 오버레이로 열어준다. 
- 자주하는질문은 페이지에 6개 항목씩 나타내며 6건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.
- 각 항목을 클릭하면 자주하는질문 수정 프레임을 오버레이로 열어주고 아이콘, 발생 원인요약, 발생 원인내용, 즉시해결, 문제가 지속 될 경우 등 Column를 수정 하도록 하고 표기 되지 않는 내용으로는 최종수정자를 관리 하도록 Database에 반영하고, 하단에 수정 버튼을 배치한다.
- 질문추가 버튼을 클릭하는 경우는 자주하는질문 추가 프레임을 오버레이로 열어 주고 아이콘, 발생 원인요약, 발생 원인내용, 즉시해결, 문제가 지속될 경우 등 Column를 추가 가능하도록 하고 표기 되지 않는 내용으로는 작성자를 관리 하도록 Database에 반영하고, 하단에 추가 버튼을 배치한다.
- 질문삭제 버튼은 해당 항목을 클릭한 후 삭제 되도록 하고 메시지로 다시한번 삭제 의사를 물어 보는 절차를 소스에 반영한다.
12-7. 일반문의 List 관리 프레임
: 일반 문의 답변 List의 Column은 문의일시, 문의제목, 문의자, 답변일시, 답변자, 잠금아이콘 등을 오버레이 프레임으로 나타낸다.
- List에 답변일시 Column에 값이 있는 경우는 수정, 삭제 버튼을 활성화 하고, 답변일시 Column에 값이 없는 경우는 답변하기 버튼을 활성화 한다. 
- 우측 List 제목 위쪽에 검색 기간을 배치하고 기본값은 금일 기준이고, 선택 된 검색 기간내에 조회하도록 List에 반영한다.
- List의 Record가 10건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.
- 우측 List 제목 위쪽에 "미 답변만 조회" 버튼을 배치하고 클릭 시 답변일시 Column에 값 없는 Record 만 List에 나타낸다.
- 좌측 상단에 새로고침 버튼을 배치하고 클릭 시 초기화 하여 List를 현재기준으로 나타낸다.
12-8. 답변하기 프레임
: 일반 문의 답변 List에서 답변하기 버튼을 클릭하면 배정담당자의 일반문의사항 페이지의 "10-8. 답변하기 프레임", "10-9. 답변하기에 입력 되는 Column"과 같이 오버레이로 답변하기 프레임을 열어주고 같은 절차로 처리한다.
12-9. 답변수정하기 프레임
: 일반 문의 답변 List에서 수정 버튼을 클릭하면 "10-8. 답변하기 프레임"과 같이 조회 정보를 나타내고 "10-9. 답변하기에 입력 되는 Column"의 Column과 같이 수정 할 수 있는 답변수정하기 프레임을 열어 준 후 하단에 배치 된 수정하기 버튼을 클릭하면 답변일시 Column 값을 현재시점으로 변경한다.
12-10. 답변삭제하기 프레임
: 일반 문의 답변 List에서 삭제 버튼을 클릭하면 문의일시, 문의자, 문의내용, 답변내용, 답변자 Column 등의 조회 정보를 오버레이로 답변삭제하기 프레임 열어주고 하단에 배치 된 삭제하기 버튼을 누르면 삭제한다.
12-11. 관리매니저 정보변경
: 관리매니저 페이지 우측 메뉴 헤더에 배치 된 "정보변경"은 사용자의 회원정보를 수정하도록 구성한다. 반드시 회원정보 수정에서는 비밀번호 변경 시 기존 비밀번호 확인 후 변경하도록 구성한다.

- 배정(h)는 "배정(h) = 배정일시(assign_date 컬럼) - 신청일시(request_date 컬럼)"계산하여 그 차를 표기한다. 예) 차의 값이 30분인경우 0.5로 표기하고 값이 없을 경우 "-"만 표기(특히. 2개 컬럼 중 1개이상의 값의 존재 하지 않는 경우도 "-"만 표기)
- 작업(h)는 "작업(h) = 작업완료일시(work_complete_date 컬럼) - 작업시작일시(work_sart_date컬럼)"계산하여 그 차를 표기한다. 예) 차의 값이 30분인경우 0.5로 표기 값이 없을 경우 "-"만 표기(특히. 2개 컬럼 중 1개이상의 값의 존재 하지 않는 경우도 "-"만 표기)






13. 시스템관리 페이지
13-1. 레이아웃 구성
: 시스템 전체를 관리 하는 책임자로 전체 업무를 관리 할 권한을 가지며, 시스템관리 페이지는 3가지 프레임으로 구성한다.
        프레임                 구성
- 서비스 집계현황         ← 전체 기준 서비스 집계 현황 그래프
- 서비스선택                ← 4가지 프레임 이동 항목(서비스 작업 List 관리, 서비스 현황 리스트, 일반문의 List 관리, 사용자관리)
- 일반문의 현황            ← 일반문의 현황 집계 그래프
13-2. 서비스 작업 List 관리 프레임
: 시스템관리는 서비스 작업 List 관리 작업 권한은 전체 기준의 관리 권한을 가진다.
- 초기의 List에 나타낼 Record는 관리매니저의 "12-2. 서비스 작업 List 관리 프레임"과 같지만 두번째 조건의 조치담당자 현황은 전체기준으로 List를 나타낸다.
- 그외 작업 절차는 "12-3. 배정작업 및 재배정작업", "12-4. 작업정보관리 프레임", "12-5. 작업정보삭제 프레임" 와 같다.
13-3. 서비스현황 리포트 프레임
: 서비스현황을 리포트 형식으로 조회 할 경우 사용 하도록 하고 List 제목 상단에 검색조건을 추가하여 검색조건에 맞게 리포트 되도록 구성한다.
- 초기는 전체 기준으로 현재 시점으로 List를 나타내고 검색조건에 따라 적용한다. 
- List에 표기되는 Column는 신청번호, 신청제목, 현재상태, 신청자, 신청소속, 단계, 배정(h), 서비스유형, 조치담당자, 조치담당자소속, 작업(h) 등을 나타낸다.
- 배정(h)는 "배정(h) = 신청시간 - 배정시간"계산하여 그 차를 표기한다. 예) 차의 값이 30분인경우 0.5로 표기
- 작업(h)는 "작업(h) = 작업시작시간 - 작업완료시간"계산하여 그 차를 표기한다. 예) 차의 값이 30분인경우 0.5로 표기
- 검색 조건에는 기간, 현재상태, 부서, 단계로 구성하여 검색조건에 맞게 List를 나타낸다.
- 하단에 자료 다운로드 버튼을 배치하고 클릭하면 자료를 로컬에 다운로드 되도록 하며, 다운로드 시 신청일시, 배정일, 예상조율일시, 작업시작일시, 작업완료일시 등이 포함 되어 다운로드 되도록 구성한다.
- 좌측 상단에 새로고침 버튼을 배치하고 클릭 시 초기화 하여 List를 현재기준으로 나타낸다.
13-4. 일반 문의 List 관리 프레임
: 시스템관리는 일반 문의 List 관리 작업 권한은 관리매니저와 같이 관리 권한을 가진다.
- 프레임 표시 형식및 작업 절차는 관리매니저의 "12-7. 일반문의 List 관리 프레임", "12-8. 답변하기 프레임", "12-9. 답변수정하기 프레임"와 같다.
13-5. 사용자관리 프레임
: 시스템에 등록 된 모든 사용자 및 기술자는 사용자 관리에서 관리되며 그 권한은 시스템관리에서 조정 관리한다.
- 사용자 관리 프레임의 List는 이메일, 이름, 소속, 직급, 권한, 생성일시, 상태 등으로 Column를 나타내고 각 Record에는 수정버튼과 초기화 버튼을 배치한다.
- List 제목 상단 에는 부서, 기간, 성명찾기 검색조건을 배치하여 검색조건에 따라 List를 적용한다.
- List의 Record가 10건 이상일 경우 하단에 다음보기 버튼을 활성화 한다.
- 좌측 상단에 새로고침 버튼을 배치하고 클릭 시 초기화 하여 List를 현재기준으로 나타낸다.
- 수정 버튼을 클릭하면 해당 Record의 회원정보를 수정하도록 오버레이로 회원정보수정 프레임을 열어주고 수정 시 반영한다.
- 초기화 버튼을 클릭하면 해당 사용자의 비밀번호을 초기화 하는 절차를 진행하고 임시비밀번호을 발급하여 보안이 적용되는 절차로 적용한다.
 13-6. 시스템관리 정보변경
: 시스템관리 페이지 우측 메뉴 헤더에 배치 된 "정보변경"은 사용자의 회원정보를 수정하도록 구성한다. 반드시 회원정보 수정에서는 비밀번호 변경 시 기존 비밀번호 확인 후 변경하도록 구성한다.
